正文#

Maven简介#

老规矩，先快速浏览官方文档，有一个大致的了解。

Apache Maven is a software project management and comprehension tool. Based on the concept of a project object model (POM), Maven can manage a project’s build, reporting and documentation from a central piece of information.

Maven开发流程

  1. 新建Maven项目

  2. 在中央仓库(也有一些第三方仓库)查找第三方jar的依赖文本

  3. 拷贝依赖文本至项目的pom.xml

  4. 执行maven build，编译/构建整个项目

为什么要用Maven#

从切身经验上来说，管理dependencies是一件很dirty的事情，尤其是停留在通过绝对路径来导入模块的程度的话，只要移动了相关文件，就需要重新手动调整。

其次，很多时候我们会在 Github上下载一些别人的project来参考学习，如果该项目没有用构建工具的话，你会为补充依赖项，以及适配版本花费大量精力。

Maven实操（IDEA）#

File >> New >> Project...，选择Maven即可，其下方的Gradle也是同类的构建工具。

以javax.mail为例，选择 usages 较大的版本（意味着稳定，一般不选择非常新的版本）。

将dependency信息复制到pom.xml文件的<dependencies></dependencies>块中

实测IDEA中Reload Project会迅速将添加到pom.xml文件中的依赖项下载

可以看到External Libraries中已经自动添加了相应的模块

此后，在IDEA中可以直接Run响应的测试模块，非常方便。
